API Reference — All Endpoints
Hono server running on port 3001. All routes require X-API-Key and X-Agent-Id headers.
🧠
Memory Endpoints
routes/memories.ts
  • POST/v1/memories — Store a memory. Body: content, type, importance, tags, associations
  • GET/v1/memories/search?q=… — Recall. Params: limit, type, tags, maxHops, blendWeight
  • GET/v1/memories/:id — Get single memory by ID
  • DEL/v1/memories/:id — Hard delete + cascade associations
  • POST/v1/memories/:id/associate — Explicit link. Body: targetId, strength, origin
📊
Dashboard / WebSocket
routes/dashboard.ts
  • WS/v1/dashboard/stream — WebSocket. Query: apiKey, agentId, type filter
  • GET/health — Health check → {"status":"ok"}
Events pushed over WS: memory:created, memory:accessed, memory:deleted, association:created. Ping/pong heartbeat every 30s.
🛂
Memory Passport
routes/passport.ts
  • POST/v1/export — Serialize agent memory. Query: format=json (default: gzip+AES-256-GCM)
  • POST/v1/import — Ingest passport. Re-embeds if model differs. Conflict: skip / merge / overwrite
Export envelope format: 1mbrain.passport.envelope v1 — includes IV, authTag, gzip payload, source agent metadata.
☁️
Google Drive Backup
routes/backup.ts
  • GET/v1/auth/gdrive — OAuth2 authorization URL
  • GET/v1/auth/gdrive/callback — Token exchange callback
  • POST/v1/backup/gdrive — Push encrypted snapshot. Filename: 1mbrain-backup-{agent}-{ts}.enc
  • GET/v1/backup/gdrive — List available backup snapshots
  • POST/v1/restore/gdrive — Pull selected snapshot + import

⚡ Spreading Activation — Graph Walk
1️⃣
Vector Pass
Embed the query string → cosine similarity search against all stored embeddings → top-K results (configurable limit).
2️⃣
Graph BFS
Start from top-K seed nodes → BFS outward up to maxHops hops through association edges → collect candidate nodes with activation scores.
3️⃣
Score Blend
Final score = (1 − blendWeight) × vectorScore + blendWeight × graphScore. Nodes below activationThreshold are pruned.

⏳ Decay System — Background Loop
⏰
Interval Timer
Configurable decay.intervalMs (default every minute). Runs as background loop started on server init.
📉
Score Decay
decayScore *= (1 − decayRate) per tick. Minimum score protected by decay.minScore. Association edge strength also decays.
📈
Access Boost
On every recall, decayScore is boosted back up. Frequently-accessed memories stay fresh; unused ones decay toward minimum.
SDK & Adapters
Everything needed to integrate 1MBrain into any agent framework.

TypeScript SDK
npm install @1mbrain/sdk
import { OneMBrainClient } from '@1mbrain/sdk'; const brain = new OneMBrainClient({ apiUrl: 'http://localhost:3001', apiKey: process.env.ONEMILLION_API_KEY, agentId: 'my-agent', }); // Store const mem = await brain.remember({ content: 'User prefers dark mode', type: 'semantic', importance: 0.9, }); // Recall const results = await brain.recall({ query: 'preferences' });

from onemillionbrain import OneMBrainClient brain = OneMBrainClient( api_url="http://localhost:3001", api_key="your-api-key", agent_id="my-agent", ) # Store mem = brain.remember( "User prefers dark mode", type="semantic", importance=0.9, ) # Recall results = brain.recall("preferences", limit=5) for r in results: print(f"[{r.score:.3f}] {r.memory.content}")
🤖 Hermes-Specific Adapter
Auto-maps Hermes context → correct 1MBrain memory type. Injects source:hermes tag. Provides buildContext() for direct LLM prompt injection.
rememberTurn()
Stores conversation Q&A pair as episodic memory. Tags: conversation-turn, topics, session:{id}.
await adapter.rememberTurn({ userMessage: 'What is pricing?', assistantReply: 'Rp 150k/month', topics: ['pricing'], });
rememberPreference()
Stores user preference as semantic memory (importance: 0.85). Tags: pref:{key}.
await adapter.rememberPreference( 'language', 'Bahasa Indonesia' );
buildContext()
Recalls relevant memories and formats as a markdown block for injection into the LLM system prompt.
const ctx = await adapter .buildContext('pricing'); // ## Relevant Memories // - [Semantic] (0.92) …
🔗 Framework Integrations
LangChain (Python)
Wrap as @tool functions. remember_tool + recall_tool exposed to the agent executor.
LlamaIndex (Python)
Wrap as FunctionTool for the agent. Works with any LlamaIndex agent/query engine.
Hermes (TypeScript)
HermesMemoryAdapter — zero-config after init. Auto-types memory on every turn.
Custom Agent (any)
Direct REST API with X-API-Key + X-Agent-Id headers. Works from any language.
